属性

简单属性: 不能被分隔成更小的部分。

复杂属性: 由多个更小的部分组成。

例: 201792123可以被分隔成年份，专业号，编号。所以是复杂属性。

信息过载: Information overload，把太多信息放到一个属性里。

如果需要将上面所说的学号拆分成年份，专业号，编号来使用的话，说明发生了信息过载。

一个例子

考虑一个学生有哪些属性:

每个属性都看一看是不是简单属性:

  • 学号，是
  • 姓名，是
  • 院系名，是
  • 出生地，不是: 它可以包含国，省，市，区。所以是复合属性。
  • 生日，是: 因为可以把年月日放到一起存，用的时候现场计算即可。
  • 年龄，是
  • 性别，是
  • 电话，同下
  • QQ号，$\text{multi-value}$: 可能有大号，小号 = 有多个值，是一个列表 -> $\text{multi-value}.$

有多个值的属性可以写作: {QQ号}，{电话}

如果出生年月是1999年，年龄写的却是14岁。这种情况是不应该发生的，同时也说明出生年月和年龄有一定的关系。

我们称年龄是出生年月的$\text{derived attribute}$，译名为导出属性。（虽然个人觉得派生属性更好一点）

现在简单的总结一下所有的属性:

  • simple 简单属性
  • composite 复杂属性
  • multi value 多值属性
  • derived 导出属性
